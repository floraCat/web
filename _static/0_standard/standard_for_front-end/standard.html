力德网站前端规范全部收起高亮重点

    基本规范 HTML规范 CSS规范 JAVASCRIPT规范

    规范目的

      为提高团队协作效率, 便于后台人员添加功能及前端后期优化维护,前端开发人员必须按本文档规范进行前端页面开发； 本文档如有不对或者不合理的地方请及时提出, 经讨论决定后方可更改。

    基本准则

    • 符合web标准
    • 结构/表现/行为 分离
    • 语义化html
    • 代码书写简洁明了有序
    • 测试需兼容Firefox,Chrome,Opera,IE(ie8及以上版本)浏览器;
    • 尽可能减小服务器负载, 保证最快的解析速度。

    前端架构

    html

    • 每个站点都应该有一个整站链接目录页（统一命名为__enter.html）
    • 整站通用的模块（如头部、底部等）需独立出来，各页面再通过java来引入
    • 样式的设置尽量只用class属性；
      js行为的选择器尽量只用data-*属性；
      id属性一般由团队确立分配，不建议私自设置

    css

    • 使用less预编译css

    javascript

    • 使用requirejs技术模块化引入脚本文件
    • 脚本一般基于jQuery库来书写：pc端使用1.10.2版本，移动端使用2.2.0版本
    • 表单的数据提交统一使用ajax完成，不建议使用默认的表单控件提交
    • 每个开发成员都有各自的js命名空间来区分各自搭建的功能模块

    辅助工具

    • 使用Git版本控制工具来管理网站
    • 使用Gulp自动化打包网站

    文件规范

    通用目录结构：

    /css /imgs ---背景图标存放处 base.css ---公用base样式表（less编译生成） a.css ---页面a样式表（less编译生成） b.css ---页面b样式表（less编译生成） /images ---图片存放处 /images_demo ---测试图片存放存，发布前整文件夹删除 index.html ---首页html /js /app ---存放整站所用到的脚本文件 jquery.js base.js ---通用脚本文件 a.js ---页面a的脚本 b.js app.js ---项目的入口初始化文件 require.js ---requirejs /less ---存放css预处理文件 var.less ---less变量 base.less ---公用base样式 mixins.less ---重用模块样式 a.less ---页面a样式 b.less ---页面b样式

    文件命名规则：

    • 名称英文字母全为小写
    • 必要时用下划线'_'作为分隔符
    • 其中不得包含汉字、空格和特殊字符
    • 一般文件以字母开头，如以下划线'_'都有特殊的含义与用途
    • 命名优先考虑对应英文，可使用常规缩写形式，如英文太生僻或复杂再考虑拼音
    • 命名原则：1）使得你自己和工作组的每一个成员能够方便的理解每一个文件的意义；
                       2）当我们在文件夹中使用“按名称排例”的命令时，同一种大类的文件能够排列在一起,以便我们查找、修改、替换、计算负载量等等操作。

    图片规范

    • 在保证视觉效果的情况下选择最小的图片格式与图片质量, 以减少加载时间
    • 非首页轮播大图外的图片大小尽量限定在 70K 左右
    • 一般比较少改动的图标用背景形式呈现，并分类合并，以减小http请求
    • 图片格式仅限于gif || png || jpg
    • 透明背景图标用png24,不透明背景且图形扁平色彩单一的用png8，否则用jpg

    html基本结构

    PC端：

    <!DOCTYPE HTML> <html lang="zh-cn"> <head> <meta charset="utf-8"> <meta name="copyright" content="LeederEducation"> <meta name="description" content=""> <meta name="keywords" content=""> <title>标题</title> <link rel="stylesheet" href="css/base.css"> <script src="js/jquery.1.10.2.min.js"></script> <script src="js/base.js"></script> </head> <body> ... </body> </html>

    移动端：

    <!DOCTYPE HTML> <html lang="zh-cn"> <head> <meta charset="utf-8"> <meta name="copyright" content="LeederEducation"> <meta name="description" content=""> <meta name="keywords" content=""> <meta name="viewport" content="width=device-width,height=device-height,inital-scale=1.0,maximum-scale=1.0,user-scalable=no"> <title>标题</title> <link rel="stylesheet" href="css/base.css" /> <script src="js/jquery.2.2.0.min.js"></script> <script src="js/base.js"></script> </head> <body> ... </body> </html>

    html格式

    • 用两个空格代替tab键实现缩进（这是唯一能保证在所有环境下获得一致展现的方法）
    • 尽量避免使用实体，如：
      • &nbsp; | &mdash; | &rdquo; 等
    • css代码不建议写在html里;如外调style文件请在head中引入
    • js代码不建议写在模块中间，要么写在head，要么写在body后面；一般情况都写在外调的js文件中；如外调js文件，其引入位置参看如下：
      • 如要兼容ie低版本浏览器，推荐： ------------------------- <html> <head> <link rel="stylesheet" href="main.css"> </head> <body> <!-- body goes here --> <script src="main.js" async></script> </body> </html> 如主要兼容现代浏览器，推荐： ------------------------- <html> <head> <link rel="stylesheet" href="main.css"> <script src="main.js" async></script> </head> <body> <!-- body goes here --> </body> </html>

    html标签

    • 页面title不可缺少
    • 不能省略可选的结束标签，如： </li>,</body>
    • 不建议使用table来布局
    • input的标题及控件用label包裹，属性for要与input的id值一致，这样点击标题时对应input才被选中或获得焦点，如： <label for="name">姓名：<input type="text" id="name" /></label>
    • 在一段完整的文字中请尽量不要使用 br/ 来人工干预分段

    html属性

    • 标签属性值使用双引号，不要使用单引号
    • ID属性在一个页面内是唯一的
    • 外调文件如是默认类型不需要声明type，如：
      • //script不用加属性type="text/javascript" <script src="js/base.js"></script> //style不用加属性type="text/style" <link rel="stylesheet" href="css/base.css" />
    • 一般情况下不使用内置style，如不推荐： <p style="color:#ddd">ppp</p>
    • 合理设置链接的target属性，如：target='_blank' | target='_self' 等

    html注释

    • 每个大模块都应该注释其栏目名称或相关说明，格式如下：
      • <!--@ 新闻中心 -->//按此加上@，以方便栏目检索
    • 一些整站通用的模块如头部/底部/侧栏等为了更好地区分开来，需要用一对对应的注释包裹起来，统一格式如下：
      • <!--@ 头部 --> <div class="pageHead"> ... </div> <!--@ /头部 -->
    • 嵌套太深或代码太多不好区分各自结束符时，应在容易混淆的结束符后添加注释如下：
      • <div class="main"> ... </div><!-- /main -->//不需加@

    html优化

    • h1 ~ h6 （每个页面有且只有一个h1,而h2~h6按层级使用）
    • img标签需要加alt属性 (装饰性的图片或图标不加alt)
      • <img src="图片.gif" alt="给图片提示">
    • a链接需要加title属性
      • <a href="#" Title="给链接文字提示">文字</a>
    • 页面元信息name为keywords/description等在发布前应添加合适内容

    css要求

    • 样式都用class设置，慎用id
    • 值为0时后面不加单位
    • 颜色值都用小写
    • 属性值用单引号，而不是双引号 （推荐：content:'aaa';）
    • 每个样式属性后加 ";"
    • 尽量用缩写，如：
      • background | border | margin | padding 等
    • 需注释所属栏目或模块标题，跟html对应模块标题一致; 多个模块标题用'&'分隔
    • 相同或类似的模块尽量把代码整合重用起来
    • css慎用expression表达式

    z-index层叠数

    • 默认的z-index值：
          * fixed顶部(90)
          * 覆盖层h_mask(99)
          * 弹窗(990)
    • 值不能超过9999
    • 除了以上模块以外需要设置z-index的，只能在11~88之间的数值中选择

    css命名

    • 不要用有唯一指定性的类名，转用能反应其目的和用途的名称，如下例:
      • 不推荐： ----------- .fw_800{ font-weight:800;} .red{ color:red;} 推荐： ----------- .heavy{ font-weight:800;} .important{ color:red;}

    js要求

    • js用data-*作为选择器，不用class & id
    • 总是使用var声明变量，且要求变量集中在函数开头声明, 避免全局变量的污染
    • 使用闭包时防止循环引用

    js格式

    • 用两空格缩进
    • 每句语句后加';'（方便压缩工具"断句"）
    • if、while、for、do语句的执行体总是用"{"和"}"括起来，即使在其结构体内只有一条语句
    • 应该优先使用单引号(‘)，因为字符串可能包含带有双引号(“)的HTML代码

    js命名规则

    • 变量命名: 以下划线'_'开头，驼峰式命名, 首字母须小写, 如_iTaoLun
    • 函数命名: 首字母小写驼峰式命名. 如iTaoLun()
    • 类命名: 首字母大写, 驼峰式命名. 如 ITaoLun
    • 常量（不会重定义的变量）：全部大写，下划线'_'分隔

    尽量避免

    • 尽量避免使用存在兼容性及消耗资源的方法或属性, 比如eval() & innerText
    • 避免对象上使用with()
    • 避免数组使用for in

    js注释

    • 注释尽量详尽且到位
    • 需写明所属页面或栏目
    • 类等脚本文件需添加头描述：
      • /* * 简述 * * 功能详细描述 * * @param <String> arg1 参数1 * @param <Number> arg2 参数2，默认为0 * @return <Boolean> 看xxx是否成功 */ function fooFunction (arg1, arg2) { }
